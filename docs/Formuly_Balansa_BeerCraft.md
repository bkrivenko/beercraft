# Формулы баланса — BeerCraft

**Версия:** 1.0 · Приложение к ТЗ BeerCraft

Документ описывает расчётную модель игры **BeerCraft**: как из рецепта и точности мини-игры получаются параметры пива, оценка качества, цена и опыт. Формулы упрощают реальную пивную науку (Tinseth, Morey, стандартный расчёт ABV) до игрового уровня. **Все коэффициенты вынесены в `game-config`** и подбираются при балансировке; расчёты выполняются **на сервере**.

## B.0. Единицы и обозначения

| Обозн. | Что | Ед. |
|---|---|---|
| `V` | объём партии | литры (игровые) |
| `amount_malt` | масса солода | кг |
| `ppkg` | потенциал солода (gravity points на кг·л) | конфиг |
| `color_L` | цвет солода | °Lovibond |
| `mash_eff` | эффективность затирания | доля (0–1), зависит от оборудования |
| `amount_hop` | масса хмеля | г |
| `alpha` | альфа-кислоты хмеля | доля (напр. 0.12 = 12%) |
| `U` | утилизация хмеля (по тайм-слоту) | доля |
| `a` | аттенюация дрожжей | доля (0.65–0.85) |
| `Acc_x` | точность этапа мини-игры x | 0–1 |

> Конвенция плотности: «gravity points» `GP` — это дробная часть OG ×1000 (например `GP = 62` ⇔ `OG = 1.062`).

---

## B.1. Начальная плотность (OG)

```
GP = ( Σ_i  amount_malt_i × ppkg_i ) × mash_eff / V
OG = 1 + GP / 1000
```
`ppkg` (конфиг, ориентир): базовые солода ≈ 300, специальные ≈ 250–290.

**Пример** (V = 20 л, mash_eff = 0.80):
Pale 4.5×300=1350; Caramel40 0.4×260=104; Munich 0.3×290=87 → Σ = 1541.
`GP = 1541 × 0.80 / 20 = 61.6` → **OG ≈ 1.062**.

---

## B.2. Конечная плотность (FG) и влияние затирания

Температура затирания смещает сбраживаемость (ниже t° → суше, выше t° → полнее):

| Темп. затирания | `mashTempFactor` |
|---|---|
| 63–64 °C | 1.05 |
| 65–67 °C | 1.00 |
| 68–70 °C | 0.93 |

```
eff_att = clamp( a × mashTempFactor × (0.97 + 0.03 × Acc_mash), 0.50, 0.92 )
FG = 1 + (OG − 1) × (1 − eff_att)
```
`Acc_mash` — точность этапа затирания (мини-игра): чем точнее держал температуру, тем ближе к целевой сбраживаемости.

**Пример** (a = 0.78, 66 °C → factor 1.00, Acc_mash = 0.90):
`eff_att = 0.78 × 1.00 × (0.97 + 0.027) = 0.78 × 0.997 ≈ 0.777`
`FG = 1 + 0.062 × (1 − 0.777) = 1 + 0.062 × 0.223 ≈ 1.0138` → **FG ≈ 1.014**.

---

## B.3. Крепость (ABV)

Стандартная формула:
```
ABV(%) = (OG − FG) × 131.25
```
**Пример:** `(1.062 − 1.014) × 131.25 = 0.048 × 131.25 ≈ 6.3 %`.

---

## B.4. Горечь (IBU)

Упрощённая модель на основе Tinseth. Для каждой засыпки хмеля `i`:
```
mgL_i = amount_hop_i × alpha_i × 1000 / V          (мг/л альфа-кислот)
IBU_i = U_i × gravityFactor × mgL_i × Acc_hops_i
IBU   = Σ_i IBU_i
```
Утилизация по тайм-слоту (конфиг):

| Момент внесения | `U` | Эффект |
|---|---|---|
| Горечь (60 мин) | 0.21 | максимум IBU |
| Вкус (15–20 мин) | 0.12 | часть IBU + вкус |
| Аромат (1–5 мин / flameout) | 0.05 | мало IBU, много аромата |
| Сухое охмеление | 0.00 | только аромат |

`gravityFactor` (плотный сусло хуже извлекает горечь): `gravityFactor = 1.65 × 0.000125^(OG−1)` (Tinseth «bigness»); для игры можно аппроксимировать таблицей ~0.95–1.0 при OG 1.05–1.07.

`Acc_hops_i` — попал ли игрок в тайм-слот в мини-игре (промах → реализованный IBU/аромат снижается).

**Пример** (V = 20, gravityFactor ≈ 0.95, Acc_hops = 1.0):
- Magnum 30 г, α 0.13, 60 мин: `mgL = 30×0.13×1000/20 = 195`; `IBU = 0.21×0.95×195 = 38.9`
- Cascade 25 г, α 0.06, 15 мин: `mgL = 75`; `IBU = 0.12×0.95×75 = 8.6`
- Citra 30 г, α 0.12, 5 мин: `mgL = 180`; `IBU = 0.05×0.95×180 = 8.6`
- Citra 40 г сухое охмеление: `IBU = 0`
→ **IBU ≈ 56** (в стиле American IPA 40–70).

---

## B.5. Цвет (SRM) и привязка к графике

```
MCU = ( Σ_i  color_L_i × amount_malt_i ) / V
SRM = srm_k × MCU^0.69            (srm_k — конфиг, ≈ 5.0)
```
**Пример:** Pale 2×4.5=9; Caramel40 40×0.4=16; Munich 9×0.3=2.7 → Σ=27.7; `MCU = 27.7/20 = 1.385`; `SRM = 5.0 × 1.385^0.69 ≈ 5.0 × 1.25 ≈ 6.3`.

**SRM → цвет жидкости** (для параметрической отрисовки `BeerCard`, ориентир):

| SRM | Цвет (hex) | Описание |
|---|---|---|
| 1–3 | `#F8E08E` | соломенный |
| 4–6 | `#F0C24B` | золотистый |
| 7–10 | `#D98C28` | янтарный |
| 11–17 | `#A85419` | медный/коричневый |
| 18–25 | `#5A2D10` | тёмно-коричневый |
| 26–40+ | `#1F0E07` | почти чёрный |

Мутность жидкости (haze) — из флокуляции дрожжей и стиля (NEIPA/Witbier → высокая).

---

## B.6. Профиль вкуса (вектор тегов)

Профиль — словарь `flavor[tag] → интенсивность (0–100)`. Теги: `солодовость, карамель, жжёность, фруктовость, цитрус, хвоя, тропики, пряность, кислинка, тело, сладость, чистота`.
```
flavor[tag] = normalize( Σ_ingredients  contribution_ingredient[tag] )
```
- Солода дают солодовость/карамель/жжёность (по типу).
- Хмель — цитрус/хвоя/тропики (по сорту и доле позднего охмеления).
- Дрожжи — фруктовость/пряность (эфиры), масштабируются отклонением темп. брожения (см. B.7).
- Добавки — целевые теги (кофе, фрукты, лактоза → тело/сладость).

---

## B.7. Точность мини-игры → влияние на расчёт

Каждый активный этап даёт точность `Acc_x ∈ [0,1]`:
- `Acc_mash` — затирание (B.2: смещает сбраживаемость/тело).
- `Acc_hops` — охмеление (B.4: реализованный IBU и аромат).
- `Acc_chill` — охлаждение (штраф офф-флейворов при промахе).
- `Acc_ferm` — режим брожения (корректность эфиров; при отклонении растёт `penalty_offflavor`).

Сводная точность процесса:
```
processAccuracy = w_m·Acc_mash + w_h·Acc_hops + w_c·Acc_chill + w_f·Acc_ferm
                  (Σ весов = 1; дефолт: 0.30 / 0.30 / 0.15 / 0.25)
```
Штраф офф-флейворов:
```
penalty_offflavor = k_off × ( (1−Acc_chill) + (1−Acc_ferm) )      (k_off — конфиг)
```
Оборудование расширяет «зелёные зоны»/упрощает этапы → при том же навыке `Acc_x` выше.

## B.8. Соответствие стилю (styleMatch, 0–100)

Для каждого ключевого параметра `p` стиля задан диапазон `[min_p, max_p]` (центр `mid_p`). Параметрный балл:
```
если min_p ≤ value_p ≤ max_p:   score_p = 100 − bonus_center·|value_p − mid_p|/halfRange_p
                                 (≈100 в центре, чуть ниже у краёв)
иначе (вне диапазона):           dist = расстояние до ближайшей границы
                                 score_p = max(0, 100 − k_out · dist / rangeWidth_p)
```
```
styleMatch = Σ_p ( weight_p × score_p ) / Σ_p weight_p
```
Параметры и веса (дефолт): ABV 0.25, IBU 0.25, SRM 0.15, FG/тело 0.15, ключевые теги профиля 0.20. Для «свободного стиля» (без цели) `styleMatch` не считается — оценка опирается на баланс и точность.

---

## B.9. Баланс (BU:GU)

```
BUGU = IBU / GP
```
Каждому стилю сопоставлен целевой диапазон BU:GU (например, сухой стаут и IPA — высокий; бок — низкий). Отклонение даёт небольшой штраф/бонус:
```
balanceScore = 100 − k_bal × |BUGU − BUGU_target| (clamp 0–100)
```
**Пример:** `BUGU = 56/62 ≈ 0.90` — для IPA уместно → `balanceScore ≈ 95`.

---

## B.10. Итоговая оценка качества (0–100)

```
Quality = clamp01to100(
      w1 × styleMatch
    + w2 × (processAccuracy × 100)
    + w3 × balanceScore
    + bonus_water           // профиль воды соответствует стилю (+, конфиг)
    + bonus_creativity       // для авторских стилей / удачных сочетаний
    − penalty_offflavor
    + random
)
```
- Дефолтные веса: `w1 = 0.5, w2 = 0.3, w3 = 0.2` (стиль важнее всего, затем мастерство, затем баланс).
- `random ∼ U(−R, +R)`, где `R = R0 × (1 − equipmentQuality)` — разброс «повезло/не повезло», **сужается** с ростом оборудования и навыка (от ±R0 до ~0).

**Пример** (styleMatch 92, processAccuracy 0.89, balanceScore 95, bonus_water +2, penalty 0, random ≈ +1):
`Quality = 0.5·92 + 0.3·89 + 0.2·95 + 2 + 1 = 46 + 26.7 + 19 + 3 = 94.7` → **≈ 95 (отличный IPA)**.

---

## B.11. Экономика: цена продажи

```
sell_price = base_price(style) × qualityMult × demandMult × reputationMult
```
- `base_price(style)` — конфиг по стилю (крепкие/сложные дороже).
- `qualityMult = 0.5 + (Quality / 100) × 1.5` → при Quality 50 = 1.0; 100 = 2.0; <40 ≈ 0.5.
- `demandMult = 0.7 … 1.5` — тренд/сезон/событие по стилю.
- `reputationMult = 1.0 … 1.3` — репутация пивоварни.

**Пример** (base 100, Quality 95 → qualityMult ≈ 1.93, demand 1.2, rep 1.1):
`100 × 1.93 × 1.2 × 1.1 ≈ 255` за партию.

Доход от **заказа** = фиксированная награда заказа при выполнении условий (стиль + диапазоны + мин. качество) + бонус репутации.

---

## B.12. Прогрессия и опыт

```
XP_за_варку = round( Quality × c_q + V × c_v + newStyleBonus )      (дефолт c_q=0.5, c_v=0.5)
XP_до_уровня(L) = base_xp × L^1.5                                   (дефолт base_xp=100)
```
**Пример:** Quality 95, V 20, новый стиль (+10): `XP = 95·0.5 + 20·0.5 + 10 = 47.5 + 10 + 10 ≈ 68`.
Достижение уровня даёт разблокировки (ингредиенты/стили/слоты — ТЗ §5.7).
