# Спецификация экранов — BeerCraft (Telegram Mini App)

**Версия:** 1.0 · Приложение к ТЗ BeerCraft

Документ описывает экраны игры **BeerCraft** для реализации в виде Telegram Mini App (WebApp). Уровень детализации — вайрфреймы + состав элементов, данные, действия и состояния. Это основа для дизайнера и фронтенд-разработчика.

## A.0. Общие принципы и платформенные ограничения

- **Mobile-first**, целевая ширина ~380 px, верстка тянется до планшета. Высота — динамическая (`Telegram.WebApp.viewportStableHeight`).
- **Telegram WebApp API** используется нативно:
  - `themeParams` — авто светлая/тёмная тема, цвета берём из темы клиента;
  - `MainButton` — главное действие экрана (крупная кнопка внизу, единый паттерн);
  - `BackButton` — навигация назад в стеке;
  - `HapticFeedback` — отклик в мини-игре варки и при важных действиях;
  - `showPopup`/`showConfirm` — подтверждения (продажа, трата ресурсов).
- **Навигация:** нижняя панель вкладок (BottomNav) + стек экранов поверх неё (конструктор, мини-игра, детали открываются как push-экраны).
- **Арт:** эстетика крафтовой пивоварни (см. ТЗ §8.2). Главный визуальный объект — `BeerCard` (карточка пива: иллюстрация + панель параметров).

### Нижняя навигация (Этап А: Solo + Дуэль)
`[ 🍺 Пивоварня ] [ 📖 Рецепты ] [ 🏪 Рынок ] [ ⚔️ Дуэль ] [ 👤 Профиль ]`

> На Этапе Б вкладка «Дуэль» расширяется в «Соревнования» (дуэль + фестиваль), добавляется доступ к кооперативу.

### Карта экранов
```
Пивоварня (Э-1, дашборд)
   ├─ Сварить → Конструктор рецепта (Э-2) → Мини-игра варки (Э-3) → Результат партии (Э-3р)
   ├─ Партия готова → Результат/забор партии
   └─ Заказы → Рынок (Э-5)
Рецепты (Э-4)  → детали рецепта / детали стиля
Рынок (Э-5)    → заказ / продажа партии
Магазин (Э-6)  ← из Пивоварни и Рынка
Дуэль (Э-7)    → лобби → задание → Конструктор (Э-2) → Мини-игра (Э-3) → Сравнение результатов
Профиль (Э-8)  → настройки, достижения, фирменные сорта
```

### Общие компоненты
- `AppHeader` — баланс (валюта), репутация, уровень/опыт.
- `BottomNav` — 5 вкладок.
- `BeerCard` — иллюстрация пива + панель параметров (ABV/IBU/цвет/профиль/оценка), бейджи.
- `ParamPanel` — строка/блок параметров со значениями и индикаторами «в стиле/вне стиля».
- `StatChip` — компактный показатель (например, «IBU 59»).
- `StyleMatchBar` — индикатор соответствия стилю (0–100%).
- `Timer` — обратный отсчёт для фоновых процессов (брожение/выдержка).
- `PrimaryAction` — связан с `MainButton` Telegram.
- `Toast`, `Sheet/Modal`, `EmptyState`, `ErrorState`, `Skeleton/Loader`.

## A.1. Экран «Пивоварня» (дашборд) — Э-1

**Назначение:** домашний экран. Статус производства, быстрый доступ к варке и продажам.

```
┌───────────────────────────────┐
│ 🍺 Моя пивоварня      ⚙        │  ← AppHeader
│ 💰 1 240   ⭐ Реп. 32   Lvl 4 │
├───────────────────────────────┤
│ Активные партии               │
│ ┌───────────┐ ┌───────────┐   │
│ │ NEIPA      │ │ Stout      │  │  ← карточки партий
│ │ Брожение   │ │ ✅ Готово  │  │
│ │ ⏳ 3ч 12м  │ │ [Забрать]  │  │
│ └───────────┘ └───────────┘   │
├───────────────────────────────┤
│ Активные заказы (2)           │
│ • Паб «Хмель»: IPA  +180 ⏳2д │
│ • Бар «Солод»: Lager +120 ⏳5д│
├───────────────────────────────┤
│ [ 🍺 Сварить пиво ]  (Main)   │
│ [ Рынок ]   [ Магазин ]       │
└───────────────────────────────┘
│ 🍺  📖  🏪  ⚔️  👤            │  ← BottomNav
└───────────────────────────────┘
```

**Данные:** баланс/репутация/уровень; список партий (стиль, статус: варка/брожение/выдержка/готово, таймер); активные заказы (заказчик, требуемый стиль, награда, дедлайн); слоты ферментеров (занято/свободно).

**Действия и переходы:** «Сварить» → Э-2; тап по готовой партии → экран результата/забор; «Забрать» → партия в склад; тап по заказу → Э-5; «Рынок» → Э-5; «Магазин» → Э-6; шестерёнка → настройки (Э-8).

**Состояния:** нет партий (EmptyState «Сварите первую партию» + CTA); все ферментеры заняты (кнопка «Сварить» подсказывает про апгрейд в магазине); партия готова (подсветка + бейдж, MainButton может меняться на «Забрать партию»).

## A.2. Экран «Конструктор рецепта» — Э-2 (ключевой)

**Назначение:** сборка рецепта (состав + процесс) с живым предпросмотром параметров и соответствия стилю.

```
┌───────────────────────────────┐
│ ‹ Конструктор рецепта         │
│ Целевой стиль: [ American IPA ▾] │
├───────────────────────────────┤
│ Предпросмотр                  │
│ ABV 6.3% │ IBU 59 │ ◑ SRM 6   │  ← ParamPanel, обновляется вживую
│ OG 1.062 │ FG 1.014           │
│ Соответствие стилю ▓▓▓▓▓▓▓░ 92%│  ← StyleMatchBar
├───────────────────────────────┤
│ [Засыпь][Хмель][Дрожжи][Вода][+]│ ← табы разделов рецепта
│ ─────────────────────────────  │
│ Засыпь (солод)                │
│  • Pale Ale 4.5 кг   [–][+]   │
│  • Caramel 40 0.4 кг [–][+]   │
│  • Munich 0.3 кг     [–][+]   │
│  [ + Добавить солод ]         │
├───────────────────────────────┤
│ Процесс                       │
│  Затирание: 66°C  [слайдер]   │
│  Кипячение: 60 мин [слайдер]  │
│  Темп. брожения: 19°C [слайдер]│
├───────────────────────────────┤
│ [ Сохранить ]   [ Варить ] Main│
└───────────────────────────────┘
```

**Разделы (табы):**
- **Засыпь:** список солодов с количеством (стиль засыпи влияет на OG/цвет/тело).
- **Хмель:** хмелевая программа — для каждой засыпки хмеля выбирается сорт, масса и **момент внесения** (горечь 60м / вкус 15–20м / аромат 1–5м / сухое охмеление). Влияет на IBU и аромат.
- **Дрожжи:** выбор штамма (тип, аттенюация, темп. диапазон, эфирный профиль).
- **Вода:** профиль воды (мягкая/жёсткая/минеральный), влияет на «попадание» в стиль.
- **«+» Добавки:** фрукты/специи/лактоза/кофе/дуб и пр.

**Живой предпросмотр:** при любом изменении пересчёт ABV/IBU/SRM/OG/FG (см. Приложение B) и `StyleMatchBar`. Параметры вне диапазона стиля подсвечиваются (красный) с подсказкой («IBU выше стиля»).

**Действия:** «Сохранить» → в книгу рецептов (с версией); «Варить» → проверка наличия ингредиентов/денег → Э-3. Если ингредиентов не хватает — Sheet «Докупить в магазине» → Э-6.

**Состояния:** пустой рецепт (подсказки-плейсхолдеры); недостаточно ингредиентов (бейдж на разделе + блокировка «Варить»); «свободный стиль» (целевой стиль не выбран → `StyleMatchBar` скрыт, доступно сохранение как авторский стиль на Этапе Б).

## A.3. Экран «Мини-игра варки» — Э-3 (ключевой)

**Назначение:** активный процесс варки с таймингом; точность исполнения влияет на качество (ТЗ §5.2.1, формулы — Приложение B §B.7).

Последовательность: **Затирание → Кипячение/Охмеление → Охлаждение** (активные мини-игры), затем **Брожение** (фоновый таймер).

### Э-3.1 Затирание (удержание в зоне)
```
┌───────────────────────────────┐
│ Этап 1/3 — Затирание          │
│ Цель: 64–67°C                 │
│  ┌─────────────────────────┐  │
│  │      ░░░██ЗОНА██░░░      │  │ ← «зелёная зона»
│  │            ▲ 66°C        │  │ ← индикатор дрейфует
│  └─────────────────────────┘  │
│  [ удерживай: нагрев / убавь ] │
│  Точность ▓▓▓▓▓▓▓░ 90%        │
└───────────────────────────────┘
```
Игрок корректирует нагрев, удерживая индикатор в зоне. Накапливается «точность этапа».

### Э-3.2 Кипячение / Охмеление (поймай момент)
```
┌───────────────────────────────┐
│ Этап 2/3 — Охмеление          │
│ Тайм-слоты внесения хмеля:    │
│ 60м[•]   15м[ ]   5м[ ]       │ ← вносить в нужные слоты
│  ⌛ 18:42 → следующий слот     │
│  [ Внести хмель ]             │
│  Точность ▓▓▓▓▓░░ 85%         │
└───────────────────────────────┘
```
Попадание в тайм-слот реализует горечь/аромат соответствующей засыпки; промах снижает реализованный IBU/аромат.

### Э-3.3 Охлаждение (тайминг-бар)
```
┌───────────────────────────────┐
│ Этап 3/3 — Охлаждение         │
│  Останови в зелёной зоне:     │
│  ▮▮▮▮▮██▮▮▮▮▮  [ Стоп ]       │
│  Точность ▓▓▓▓▓▓▓ 95%         │
└───────────────────────────────┘
```
Промах → штраф «офф-флейворы».

### Э-3.4 Брожение (фон) и Э-3р Результат
- Брожение идёт по таймеру в фоне (карточка партии на Э-1, push-уведомление при готовности). Удержание темп. режима — упрощённый авто/проверка, отклонения дают штраф.
- **Экран результата партии (Э-3р):**
```
┌───────────────────────────────┐
│ Партия готова!                │
│   [ BeerCard: American IPA ]  │ ← иллюстрация + параметры
│   Оценка качества: 92 ⭐       │
│   ABV 6.3 │ IBU 59 │ SRM 6    │
│   Точность: затир 90 / хмель 85│
│   Соответствие стилю 92%      │
│   +56 XP                      │
│ [ В склад ]  [ Продать ] Main │
└───────────────────────────────┘
```

**Состояния/edge cases:** пауза/потеря связи → этап завершается с усреднённой точностью, прогресс не теряется; повтор (идемпотентность) — результат фиксируется на сервере один раз; в дуэли «дрейфы» детерминированы общим сидом.

## A.4. Экран «Рецепты и стили» — Э-4

**Назначение:** книга рецептов игрока и каталог стилей.

```
┌───────────────────────────────┐
│ [ Мои рецепты ][ Стили ][ Сообщество ] │ ← табы
├───────────────────────────────┤
│ 🔎 Поиск / фильтр (семейство) │
│ • Моя NEIPA v3   ★92  [Варить]│
│ • Сухой стаут v1 ★88  [Варить]│
│ • American IPA   (стиль)  →   │
└───────────────────────────────┘
```

- **Мои рецепты:** карточки с версиями (v1/v2/v3), лучшей оценкой, кнопкой «Варить» (→ Э-3 по сохранённому процессу) и «Открыть» (→ Э-2 для правки).
- **Стили:** реальные стили (каталог §6 ТЗ) + авторские (Этап Б). Детали стиля: диапазоны ABV/IBU/SRM, описание, профиль.
- **Сообщество (Этап Б):** публичные рецепты/стили, автор, «Сварить по чужому стилю».

## A.5. Экран «Рынок и заказы» — Э-5

**Назначение:** выполнение заказов и продажа партий.

```
┌───────────────────────────────┐
│ [ Заказы ][ Продажа ][ Тренды ] │
├───────────────────────────────┤
│ Заказ: Паб «Хмель»            │
│ Нужен: IPA, IBU 40–70, кач.≥80│
│ Награда: 💰180 + ⭐5  ⏳ 2 дня │
│ Подходящие партии: NEIPA ✓    │
│ [ Выполнить ] Main            │
└───────────────────────────────┘
```

- **Заказы:** требования (стиль/диапазоны/мин. качество), награда, дедлайн; система подсвечивает подходящие партии со склада; «Выполнить» → списание партии, начисление награды/репутации.
- **Продажа:** выбрать партию → расчёт цены (см. Приложение B §B.11: качество × спрос/тренд × репутация) → подтверждение.
- **Тренды:** какие стили в спросе сейчас (сезонность/события) — множители цены.

## A.6. Экран «Магазин» — Э-6

```
┌───────────────────────────────┐
│ [ Ингредиенты ][ Оборудование ][ Косметика ] │
├───────────────────────────────┤
│ Цитра (хмель)  α 12%  💰40/100г│
│   [ Купить ]                   │
│ Ферментер +1 слот   💰800      │
│   [ Купить ]                   │
│ Премиум-этикетки    🔒 Скоро   │ ← косметика выключена на старте
└───────────────────────────────┘
```

- **Ингредиенты:** покупка за игровую валюту; редкие — разблокировка по уровню/наградам.
- **Оборудование:** котёл (объём), охлаждение (меньше брака/шире «зелёные зоны»), ферментеры (параллельные партии), фильтрация/розлив, склад/выдержка.
- **Косметика:** на старте показывается как «Скоро» (монетизация выключена, заложена — ТЗ §12); вкладка и карточки готовы к включению через feature-flag.

## A.7. Экран «Дуэль» — Э-7 (Этап А)

**Назначение:** соревнование 1×1.

```
ЛОББИ                          ЗАДАНИЕ
┌───────────────┐   ┌───────────────────┐
│ ⚔️ Дуэль       │   │ Задание дуэли      │
│ [Создать]      │   │ Стиль: Pilsner     │
│ [По ссылке]    │   │ Бюджет: 💰500      │
│ [Быстрый поиск]│   │ Время: 5 мин       │
│ Соперник: …    │   │ [ Начать ] Main    │
│ Готов ✅/⬜     │   └───────────────────┘
└───────────────┘
СРАВНЕНИЕ
┌───────────────────────────────┐
│ Результаты дуэли              │
│ Вы:      Pilsner  кач. 88 ✓   │
│ Соперник: Pilsner кач. 84     │
│ 🏆 Победа! +рейтинг           │
│ [ BeerCard ] vs [ BeerCard ]  │
└───────────────────────────────┘
```

- **Лобби:** создать комнату/войти по ссылке (через бота)/быстрый поиск; статусы готовности.
- **Задание:** общий стиль/бюджет/время, общий сид (честность); кнопка «Начать» ведёт в Э-2 → Э-3.
- **Сравнение:** оценки качества обоих, победитель, изменение рейтинга; карточки пива рядом. Поддержка асинхронной дуэли (соперник варит позже — экран ожидания/уведомление).

> Этап Б: добавляются экраны «Фестиваль» (лобби 2–4, турнирная таблица, сезоны/рейтинг) и «Кооператив» (общая пивоварня, роли/права, лог действий) — структурно повторяют лобби/результаты.

## A.9. Сводка состояний (для всех экранов)

| Состояние | Поведение |
|---|---|
| Loading | Skeleton/Loader, без «прыжков» layout |
| Empty | EmptyState с пояснением и основным CTA |
| Error | ErrorState + «Повторить»; ошибки сети не теряют ввод |
| Offline / разрыв | Мини-игра — усреднённая точность; действия — повтор по восстановлении (идемпотентно) |
| Недостаточно ресурсов | Блок действия + быстрый переход в магазин |
| Процесс идёт (варка/брожение) | Таймеры на Э-1, push при готовности |

## A.10. Соответствие Telegram WebApp (для разработки)

| Экран | MainButton | BackButton | Особое |
|---|---|---|---|
| Э-1 Пивоварня | «Сварить пиво» (или «Забрать партию», если готова) | скрыт | haptic при готовности |
| Э-2 Конструктор | «Варить» | назад в стек | live-пересчёт, валидация |
| Э-3 Мини-игра | контекстное действие этапа | подтверждение выхода (теряется прогресс? — нет, усреднение) | haptic на попаданиях |
| Э-5 Рынок | «Выполнить»/«Продать» | назад | `showConfirm` на продаже |
| Э-7 Дуэль | «Начать»/«Готов» | выход из лобби | realtime-статусы |
